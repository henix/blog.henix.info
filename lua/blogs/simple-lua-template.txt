<p>　　<a href="http://code.google.com/p/slt/">slt</a>（Simple Lua Template）是一个 Lua 模板引擎。模板引擎类似 printf 的格式化字符串，根据一个模板，将一些变量“串行化”成一段文本，不过功能更强大，通常用来生成 HTML 页面。</p>
<p>　　一般的模板引擎应该实现的功能有：</p>
<ul>
<li>分支条件</li>
<li>循环，可以遍历列表</li>
<li>包含文件</li>
<li>……</li>
</ul>
<p>　　我接触到的第一个模板引擎是 Java 的 <a href="http://freemarker.sourceforge.net/">FreeMarker</a> ，发现这东西可以取代 JSP 。</p>
<p>　　slt 是我在编写这个 blog 的过程中开发的，我使用 Lua 作为这个 blog 的开发语言，自然需要一个 Lua 的模板引擎。但现有的模板引擎感觉都太复杂。</p>
<ul>
<li>Kepler 的 <a href="http://cosmo.luaforge.net/">Cosmo</a> ：用法太复杂，模板、子模板，不知道什么意思。</li>
<li><a href="http://www.savarese.com/software/ltp/">ltp</a> ：感觉上可以用，但要用一套专门的命令。</li>
<li><a href="http://code.google.com/p/leslie/">leslie</a> ：Django 风格的模板，自己实现了 if/for/include ，Lua 的模板干嘛要像 Django 一样？</li>
</ul>
<p>　　Simple Lua Template 的想法很简单。就像 jsp 可以在 HTML 中嵌入 Java 代码一样，或者像 PHP 的风格一样，slt 就是在 HTML 中嵌入 Lua 代码。</p>
<pre class="brush: xml">
&lt;span&gt;
&#35;{ if user ~= nil then }
  Hello, &#35;{= user.name }!
&#35;{ else }
  &lt;a href="/login"&gt;login&lt;/a&gt;
&#35;{ end }
&lt;/span&gt;
&#35;{include: 'footer.seg.htm' }
</pre>
<p>　　我认为这样就够用了，也够简单。欢迎试用，欢迎提建议：<a href="http://code.google.com/p/slt/">http://code.google.com/p/slt/</a> 。</p>
