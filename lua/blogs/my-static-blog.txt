<p>　　最近一直在断断续续地做这个博客，当时决定自己做一个博客的时候，突发奇想想到了用纯静态的方式。当时主要是觉得用 python 的话必须用 Google 特定的数据库操作方式，不能用普通的 SQL 之类，写出来的程序不具备通用性，所以还不如干脆纯静态。</p>

<h3>优势</h3>
<p>　　显然纯静态博客有以下这些优势：</p>
<ol>
<li>速度快。访问页面不用查询数据库，自然是快得没话说。</li>
<li>最大限度的控制。相较于用 wordpress 搭建博客，自己的是想怎么写就怎么写，也不必挖空心思地去寻找能满足自己需要的插件。（变成要找 jquery 插件了……）</li>
<li>搬迁容易。如果 GAE 被墙，我可以立即换个主页空间。现在要找一个免费的静态空间很容易，但找一个 python 或 java 的动态空间实在是难上加难。</li>
</ol>

<h3>问题</h3>
<p>　　当然也有一些问题，下面我谈谈我的实现方法：</p>
<p>1. 如何实现评论？</p>
<p>　　我的评论直接用的免费的 <a href="http://disqus.com/" class="tooltip">DISQUS.COM<span>(http://disqus.com/)</span></a>。允许匿名评论，允许 OpenID 登录。</p>
<p>2. 如何统计访问量？</p>
<p>　　统计访问量我用了 <a href="http://www.google.com/analytics/" class="tooltip">Google Analytics<span>(http://www.google.com/analytics/)</span></a>，功能很强大。Google Analytics 还可以把访问量数据导出成 csv，这样我就可以在博客上显示文章的访问量了，不过不是实时更新的。</p>
<p>3. 如何统计评论数量？</p>
<p>　　DISQUS.COM 可以将评论导出成 XML，然后再写段程序统计即可。</p>
<p>4. 如何实现搜索？</p>
<p>　　我使用的是 <a href="http://www.google.com/cse/" class="tooltip">Google 自定义搜索<span>(http://www.google.com/cse/)</span></a>。</p>

<h3>缺点</h3>
<p>　　当然纯静态博客也有一些缺点：</p>
<ol>
<li>评论系统必须依赖于外部服务。其实如果我一旦搬迁，我的所有评论将无法显示。因为 DISQUS.COM 是会验证地址的。也就是说地址一变，它就认为是不同的网站，只能重新申请一个服务，以前网站上的评论无法跟着搬迁。</li>
<li>编程较繁琐。什么东西都要自己写，既要应付 IE6 的 position fixed bug，还要用 jquery 响应键盘事件，然后还要搞清楚 RSS、robot.txt、sitemap.xml 是什么东东。当然如果你自己能 enjoy 这种对细节的完全控制的话，也就没问题。</li>
</ol>

<h3>优化</h3>
<p>　　纯静态博客的架构也很简单，根本不需要后端优化。我还是做了一些前端优化：</p>
<p>　　首先的自然是牛逼哄哄的 <a href="http://developer.yahoo.com/performance/rules.html" class="tooltip">YSlow 35条<span>(http://developer.yahoo.com/performance/rules.html)</span></a>。看看 <a href="http://www.dbanotes.net/web/web_performance_optimization.html" class="tooltip">DBA Notes 的评论<span>(http://www.dbanotes.net/web/web_performance_optimization.html)</span></a>你就知道前端优化是很重要的而又往往被忽视。</p>
<p>　　这些准则中有些还是很容易做到的，比如：</p>
<ul>
<li>Put Stylesheets at the Top：把样式表放到最上面。</li>
<li>Put Scripts at the Bottom：把脚本放到最下面。</li>
</ul>
<p>　　所以你可以看看我的页面，这两条还是做到了的。</p>
<p>　　还有传说中的 CSS Sprites，雪碧技术。不过我这个网站本来就小，图片就那么几张，还雪碧呢。咱就懒得做了，图片就让它原模原样的吧。</p>
